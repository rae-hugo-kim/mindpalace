# Planning Checklist

- [ ] Goal is stated in one sentence
- [ ] Success criteria is explicit and testable
- [ ] Scope and non-goals are listed
- [ ] Key constraints are listed (safety, repo rules, deadlines)
- [ ] Verification chosen (tests/evals/manual steps)
- [ ] Risky actions identified (if any) and approval requirement noted
- [ ] If unclear: questions ≤ 7, otherwise assumptions are recorded

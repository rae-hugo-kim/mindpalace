## 발견된 문제 (다음 cycle 후보)

### 1. 검색 dilution / 약한 매칭 — 가장 큰 위험
- **증상**: Q2, Q3, Q4가 의도한 청크를 못 찾음. 짧은 텍스트나 일반 단어에 모델이 끌리는 경향.
- **추정 원인**: (a) 1 turn = 1 chunk라 긴 turn은 1500자 평균값으로 압축됨 (avg=1183자), (b) chat 코퍼스(6803) 대비 code 코퍼스(780)가 작아 결과가 chat에 치우침, (c) 임베딩 모델이 multilingual MiniLM이라 fine-grained code semantics 약함.
- **후속 후보**: sub-chunk 분할 (문장 단위 N자), title boost, rerank, top_k 증가 + 필터.
- seed.yaml의 risk "시맨틱 검색이 표면 단어에 휘둘려"가 실측 확인됨.

### 2. 0-hit UX (AC12) 불완전
- **증상**: 의도적 0-hit 쿼리("quantum chromodynamics")도 5건 반환됨 (distance 3.4-3.6). 사용자가 "관련 없는 결과"인지 식별 어려움.
- **후속**: distance threshold 도입 또는 "low confidence" 표시. 후보 threshold ≈ 3.5 (Q3의 4.87처럼 명백히 낮은 신뢰는 경고).

### 3. 빈 청크 노이즈
- **증상**: `text=''` 청크가 **900개** (전체의 12%). Code JSONL의 tool_use/tool_result 같은 비텍스트 turn들이 빈 텍스트로 들어옴.
- **영향**: embedding 1만 번 중 900번이 거의 의미 없음, search 결과 dilution, 디스크 낭비.
- **후속**: `chunk_session`에서 `text.strip() == ""` 청크 skip. seed의 Feature 3 주의사항에도 명시되어 있던 정책 결정 시점.

### 4. 메가 conversation
- top title `"(미분류 주제 묶음)"` 1개가 **708 chunks** (9%). 한 conv 내 다양한 주제 섞임 → 임베딩 평균 효과로 검색 매칭 약화.
- **후속**: conv 내 topic shift detection (v1+ 범위), 일단 v0에선 dilution 영향만 인지.

### 5. 모델 다운로드 경고
- 매 import마다 HuggingFace `HF_TOKEN` 경고 + weight loading bar. UX 노이즈.
- **후속**: 환경변수 가이드 README 추가 또는 `transformers.logging.set_verbosity_error()` 적용.

# Kickoff Checklist

## Pre-Kickoff

- [ ] 작업 요청이 명확한가? (모호하면 먼저 명확화)
- [ ] 새 프로젝트인가, 기존 프로젝트의 기능 추가인가?

---

## Phase 0: JTBD

- [ ] 최종 사용자 정의됨
- [ ] 해결할 문제 정의됨
- [ ] 성공 기준 1-2문장으로 정리됨
- [ ] **사용자 확인 받음**

---

## Phase 1: Context

- [ ] 프로젝트 구조 파악됨 (또는 "새 프로젝트"로 확인)
- [ ] 기술 스택 확인됨 (또는 "N/A")
- [ ] 빌드/테스트 명령어 발견됨 (또는 "N/A")
- [ ] 기존 패턴 확인됨 (또는 "none")
- [ ] **추측 없이 증거 기반으로만 작성됨**

---

## Phase 2: Scope

- [ ] MUST 항목 최소 1개
- [ ] SHOULD 항목 정의됨 (없으면 "없음"으로 명시)
- [ ] MUST NOT 항목 정의됨
- [ ] OUT OF SCOPE 항목 최소 1개
- [ ] **사용자 확인 받음**

---

## Phase 3: Acceptance

- [ ] 수락 기준 최소 3개
- [ ] 각 기준이 구체적이고 테스트 가능함
- [ ] 엣지 케이스 최소 2개 정의됨
- [ ] **사용자 확인 받음**

---

## Phase 4: Backpressure

- [ ] 검증 방법 정의됨 (자동 또는 수동)
- [ ] 검증 실행 방법 명시됨
- [ ] **사용자 확인 받음**

---

## Completion

- [ ] Kickoff Summary 작성됨
- [ ] `kickoff-done` 플래그 생성됨
- [ ] 다음 단계 안내됨

---

## Quick Reference

### JTBD 질문
1. 누가 사용하는가?
2. 무슨 문제를 해결하는가?
3. 성공하면 뭐가 달라지는가?

### Scope 질문
1. 반드시 해야 하는 것? (MUST)
2. 하면 좋은 것? (SHOULD)
3. 하면 안 되는 것? (MUST NOT)
4. 나중에 할 것? (OUT OF SCOPE)

### Acceptance 질문
1. 동작한다는 걸 어떻게 증명?
2. 엣지 케이스는?
3. 실패하면 어떻게 처리?
